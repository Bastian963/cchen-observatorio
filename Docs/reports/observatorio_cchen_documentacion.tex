\chapter{Resumen Ejecutivo}
%% ════════════════════════════════════════════════════════════

El \textbf{Observatorio Tecnológico CCHEN 360°} es un sistema de inteligencia de datos diseñado para la Comisión Chilena de Energía Nuclear (CCHEN), en el marco del proyecto CORFO de fortalecimiento de aplicaciones nucleares. Su objetivo es consolidar y analizar la producción científica, técnica e institucional de CCHEN, proveyendo capacidades de vigilancia tecnológica, análisis bibliométrico, gestión de capital humano y gobernanza de datos.

\section{La CCHEN y el proyecto CORFO}

La Comisión Chilena de Energía Nuclear (CCHEN) es la institución estatal responsable de la investigación, desarrollo y aplicaciones pacíficas de la energía nuclear en Chile. Dentro del \textbf{Plan de Fortalecimiento de Aplicaciones Nucleares} financiado por CORFO (CORFO CCHEN 360°), se identificó la necesidad de contar con un observatorio de inteligencia científica y tecnológica que permita:

\begin{enumerate}[leftmargin=2cm]
  \item Monitorear y analizar la producción científica institucional de forma sistemática.
  \item Vigilar el entorno tecnológico y de financiamiento relevante para las áreas nucleares.
  \item Gestionar y visualizar el capital humano en formación.
  \item Fortalecer la gobernanza de datos institucionales.
  \item Facilitar la transferencia tecnológica y la colaboración ecosistémica.
\end{enumerate}

\section{Gestión de estado y deduplicación}

Cada script de vigilancia mantiene un archivo JSON de estado para evitar registros duplicados entre ejecuciones:

\begin{lstlisting}[style=pythonstyle, caption={Patrón de gestión de estado (arxiv\_monitor.py)}]
# Cargar IDs ya procesados
def load_state() -> set:
    if STATE.exists():
        with open(STATE) as f:
            return set(json.load(f).get("seen_ids", []))
    return set()

# Guardar estado actualizado
def save_state(seen_ids: set) -> None:
    with open(STATE, "w") as f:
        json.dump({
            "seen_ids": sorted(seen_ids),
            "updated":  datetime.datetime.now().isoformat()
        }, f)

# Solo procesar entradas nuevas
new_items = [it for it in items if it["id"] not in seen_ids]
for it in new_items:
    seen_ids.add(it["id"])
\end{lstlisting}

\textbf{Generación de IDs:}
\begin{itemize}
  \item \textbf{arXiv:} extraído de la URL (\texttt{2403.12345})
  \item \textbf{Google News:} \texttt{MD5(título + fuente)[:12]}
  \item \textbf{ANID convocatorias:} \texttt{MD5(título + url)[:12]}
  \item \textbf{IAEA INIS:} ID nativo del registro InvenioRDM
\end{itemize}

  \item \textbf{Contexto institucional dinámico:} el sistema prompt (~22.770 caracteres) se construye con la función \texttt{\_build\_assistant\_system\_prompt(ctx)} con datos reales de CCHEN: 877 publicaciones, KPIs, 48 investigadores ORCID, 26 convocatorias, top journals, financiamiento ANID, etc.
  \item \textbf{RAG con pgvector:} para cada consulta del usuario se recuperan los 5 papers más relevantes via búsqueda semántica (\texttt{semantic\_search.search()}) y se inyectan en el prompt
  \item \textbf{Reranking por perfil:} si la consulta es de medicina nuclear, se amplía el pool de candidatos y se aplica un \textit{profile bonus score} para priorizar papers relevantes
  \item \textbf{Streaming:} respuestas progresivas con \texttt{st.write\_stream()}
  \item \textbf{Memoria de sesión:} historial de conversación en \texttt{st.session\_state.messages}
  \item \textbf{Max tokens:} 2{,}048 por respuesta en modo interactivo
  \item \textbf{Logo institucional:} logo CCHEN 360° en página de login (220px) y sidebar (160px) con fallback a texto si falta el archivo
\end{itemize}

\section{¿Qué son los embeddings semánticos?}

Un \textbf{embedding semántico} (o vector semántico) es una representación numérica de un texto como un punto en un espacio vectorial de alta dimensionalidad. Textos con significados similares quedan cercanos en ese espacio; textos sobre temas distintos quedan alejados.

En el sistema, cada publicación es representada por un vector de \textbf{384 dimensiones} generado por el modelo \texttt{paraphrase-multilingual-MiniLM-L12-v2} (sentence-transformers). Este modelo fue elegido porque:
\begin{itemize}
  \item Soporta \textbf{múltiples idiomas} (español, inglés, francés, alemán, etc.)
  \item Genera vectores \textbf{normalizados} (norma L2 = 1), permitiendo calcular similitud coseno mediante producto punto simple
  \item Es \textbf{compacto} (7.3M parámetros, $\sim$480 MB) pero preciso para dominio científico
  \item Tiene un contexto de 512 tokens, suficiente para título + abstract
\end{itemize}

\begin{infobox}[Similitud coseno]
Dados dos vectores $\mathbf{u}$ y $\mathbf{v}$ normalizados:
\[ \text{similitud}(\mathbf{u}, \mathbf{v}) = \mathbf{u} \cdot \mathbf{v} = \sum_{i=1}^{384} u_i \cdot v_i \in [-1, 1] \]
Un valor de 1.0 indica textos idénticos; 0.0 indica textos sin relación semántica; valores negativos son raros en textos de un mismo dominio. En la práctica, scores $>$ 0.6 indican alta relevancia temática.
\end{infobox}

\subsection*{Fase 2: Migración a Supabase pgvector (on-demand)}

La tabla \texttt{paper\_embeddings} almacena los 877 vectores en Supabase con la extensión \texttt{pgvector}:

\begin{lstlisting}[style=sqlstyle, caption={Definición de la tabla paper\_embeddings}]
CREATE TABLE paper_embeddings (
  openalex_id  TEXT PRIMARY KEY,
  doi          TEXT,
  title        TEXT,
  year         INTEGER,
  abstract     TEXT,
  embedding    vector(384)  -- extension pgvector
);

-- Indice IVFFlat para busqueda aproximada por coseno
CREATE INDEX ON paper_embeddings
  USING ivfflat (embedding vector_cosine_ops)
  WITH (lists = 50);

-- RPC para busqueda semantica desde el cliente
CREATE OR REPLACE FUNCTION match_papers(
  query_embedding vector(384),
  match_count     int DEFAULT 5
) RETURNS TABLE(openalex_id text, title text, year int,
                abstract text, similarity float)
LANGUAGE sql AS $$
  SELECT openalex_id, title, year, abstract,
         1 - (embedding <=> query_embedding) AS similarity
  FROM paper_embeddings
  ORDER BY embedding <=> query_embedding
  LIMIT match_count;
$$;
\end{lstlisting}

\textbf{Operador \texttt{<=>}:} distancia coseno en pgvector. La similitud se obtiene como \texttt{1 - (<=>)}.

\textbf{Índice IVFFlat:} divide el espacio vectorial en 50 listas (clusters). La búsqueda aproximada es $\sim$50$\times$ más rápida que fuerza bruta, con pérdida mínima de recall para corpus $<$ 10.000 vectores.

La función principal \texttt{search(query, top\_k=5)} tiene dos rutas de ejecución:

\section{BERTopic: modelado de tópicos}

\textbf{BERTopic} es una técnica de modelado de tópicos no supervisado que combina embeddings BERT con clustering HDBSCAN y una representación de tópicos via c-TF-IDF. Opera en tres pasos:

\begin{enumerate}
  \item \textbf{Embeddings}: convierte abstracts a vectores usando sentence-transformers
  \item \textbf{Reducción dimensional}: UMAP proyecta de 384 a 5 dimensiones
  \item \textbf{Clustering}: HDBSCAN agrupa papers en tópicos coherentes; papers con baja probabilidad van al tópico $-1$ (ruido)
  \item \textbf{Representación}: c-TF-IDF extrae las palabras más representativas por tópico
\end{enumerate}

El corpus CCHEN produce \textbf{23 tópicos} (sobre 358 papers con abstract suficiente). Los tópicos dominantes son dosimetría de radiaciones, medicina nuclear, física de reactores, producción de radiofármacos, e imagenología.

\textbf{Solución recomendada:} solicitar el upgrade a Dev Tier (\url{console.groq.com/settings/billing}) o usar la API de Anthropic (Claude) que tiene límites más generosos.

  \textbf{Término} & \textbf{Definición} \\
  \midrule
  \endhead
  \textbf{OpenAlex} & Índice académico abierto con 250M+ publicaciones. Sucesor de Microsoft Academic Graph. \url{openalex.org} \\
  \textbf{CrossRef} & DOI registration agency. Provee metadatos de publicaciones, incluyendo fuentes de financiamiento. \url{crossref.org} \\
  \textbf{ORCID} & Open Researcher and Contributor ID. Identificador único persistente para investigadores. \url{orcid.org} \\
  \textbf{ROR} & Research Organization Registry. Identificadores abiertos para instituciones de investigación. \url{ror.org} \\
  \textbf{DOI} & Digital Object Identifier. Identificador persistente para publicaciones y datasets. \\
  \textbf{arXiv} & Repositorio de preprints (física, matemáticas, ciencias de la computación, etc.). \url{arxiv.org} \\
  \textbf{IAEA INIS} & International Nuclear Information System. Base de datos nuclear especializada de la IAEA. \\
  \textbf{InvenioRDM} & Plataforma de repositorio digital institucional de código abierto. Usada por IAEA INIS. \\
  \textbf{SJR} & SCImago Journal Rank. Indicador de impacto de revistas basado en Scopus. \\
  \textbf{H-index} & Índice Hirsch. Mayor $h$ tal que $h$ publicaciones han recibido al menos $h$ citas. \\
  \textbf{OA} & Open Access. Acceso abierto a publicaciones científicas sin barreras de pago. \\
  \textbf{Gold OA} & La revista es completamente de acceso abierto. \\
  \textbf{Green OA} & Copia abierta en repositorio (preprint, postprint, tesis). \\
  \textbf{Hybrid OA} & Revista de pago con opción de apertura por artículo. \\
  \textbf{Bronze OA} & Libre en el sitio del editor sin licencia explícita (temporal). \\
  \textbf{TRL} & Technology Readiness Level. Escala 1--9 de madurez tecnológica (ESA/NASA). \\
  \textbf{DuckDB} & Motor de base de datos analítica embebido. Ideal para queries sobre CSV/Parquet. \\
  \textbf{Supabase} & Backend-as-a-Service basado en PostgreSQL. Incluye API REST, autenticación y storage. \\
  \textbf{Groq} & Plataforma de inferencia LLM de alta velocidad (\url{console.groq.com}). \\
  \textbf{LLM} & Large Language Model. Modelo de lenguaje de gran escala. ej: llama-3.3-70b. \\
  \textbf{BERTopic} & Técnica de modelado de tópicos basada en BERT + clustering (HDBSCAN + c-TF-IDF). \\
  \textbf{SBERT} & Sentence-BERT. Embeddings semánticos de oraciones para NLP. \\
  \textbf{ETL} & Extract, Transform, Load. Pipeline de ingestión y transformación de datos. \\
  \textbf{GitHub Actions} & Plataforma de CI/CD integrada en GitHub. Usada para automatización semanal. \\
  \textbf{Cron} & Mecanismo de programación de tareas en Unix/Linux. Formato: \texttt{min hora día mes dow}. \\
  \textbf{DataCite} & Registro de metadatos para datasets y outputs de investigación. \url{datacite.org} \\
  \textbf{OpenAIRE} & Infraestructura europea para outputs de investigación abierta. \url{openaire.eu} \\
  \textbf{PatentsView} & API del USPTO para búsqueda y análisis de patentes. \url{patentsview.org} \\
  \textbf{Unpaywall} & Servicio que identifica copias legales abiertas de artículos de pago. \url{unpaywall.org} \\
  \textbf{Data Lake} & Almacenamiento de datos en formato raw/original (aquí: CSVs en \file{Data/}). \\
  \textbf{Data Warehouse} & BD estructurada para análisis (aquí: Supabase/PostgreSQL). \\
  \textbf{Streamlit} & Framework Python para crear aplicaciones web de datos de forma rápida. \\
  \textbf{Plotly} & Librería de visualización interactiva para Python/JavaScript. \\
  \textbf{NetworkX} & Librería Python para análisis y visualización de grafos y redes. \\
  \textbf{pandas} & Librería Python para manipulación y análisis de datos tabulares. \\
  \midrule
  \textbf{pgvector} & Extensión PostgreSQL que agrega soporte para vectores de alta dimensionalidad. Permite almacenar embeddings y realizar búsqueda por similitud (coseno, L2, producto punto) directamente en SQL. \\
  \textbf{IVFFlat} & Índice de búsqueda aproximada para pgvector. Divide el espacio vectorial en $k$ listas (clusters); la búsqueda solo recorre las listas más cercanas al query vector. Mucho más rápido que fuerza bruta con pérdida mínima de recall para corpus pequeños. \\
  \textbf{RAG} & Retrieval-Augmented Generation. Técnica que combina recuperación de documentos relevantes con generación LLM. El sistema recupera los papers más relevantes via búsqueda semántica y los inyecta en el prompt antes de generar la respuesta. \\
  \textbf{Embedding semántico} & Representación numérica de un texto como vector de alta dimensionalidad (aquí: 384 dims). Textos con significados similares tienen vectores cercanos. Generado por el modelo \texttt{paraphrase-multilingual-MiniLM-L12-v2}. \\
  \textbf{sentence-transformers} & Librería Python para generar embeddings semánticos de oraciones. Basada en Transformers (BERT, RoBERTa, etc.). Soporte multilingual. \\
  \textbf{MiniLM} & Familia de modelos de lenguaje compactos (Microsoft Research). \texttt{paraphrase-multilingual-MiniLM-L12-v2} tiene 384 dimensiones y soporta 50+ idiomas. \\
  \textbf{Similitud coseno} & Métrica de similitud entre vectores normalizados, calculada como el producto punto. Valor 1.0 = idénticos, 0.0 = sin relación semántica. Usada en búsqueda semántica. \\
  \textbf{Profile reranking} & Técnica de reordenamiento de resultados que aplica un bonus score basado en un perfil temático. En CCHEN, se usa para priorizar papers de medicina nuclear cuando la consulta es sobre ese tema. \\
  \textbf{TPD (Tokens Per Day)} & Límite de tokens diarios en APIs de LLM. Groq tier gratuito: 100.000 TPD por organización para \texttt{llama-3.3-70b-versatile}. \\
  \textbf{TPM (Tokens Per Minute)} & Límite de tokens por minuto. Groq \texttt{llama-3.1-8b-instant} tiene TPM=6.000, insuficiente para el prompt del sistema ($\sim$6.600 tokens). \\
  \textbf{Heurística} & En el contexto de evaluación del asistente: métrica calculable automáticamente (sin juicio humano) como proxy de calidad. No es una medida de verdad, sino un indicador aproximado. \\
  \textbf{heuristic\_sources\_mentioned} & Métrica heurística: cuenta cuántos nombres de fuentes esperadas aparecen como substring en la respuesta del asistente (ej: ``publicaciones'', ``convocatorias''). \\
  \textbf{heuristic\_citation\_tags} & Métrica heurística: cuenta etiquetas de citación estilo \texttt{(fuente: ...)} en la respuesta del asistente. Indicador de cuán explícitas son las referencias a fuentes. \\
  \textbf{structured\_or\_hybrid} & Modo de evaluación para consultas que requieren síntesis de múltiples tablas estructuradas (Supabase) más contexto semántico. Evaluado con respuesta LLM + métricas heurísticas. \\
  \textbf{publication\_rag} & Modo de evaluación para consultas que requieren recuperación semántica de papers. Evaluado por keyword hits, scores de similitud y cobertura de abstracts. \\
  \textbf{context\_trace} & Diccionario JSON que registra cuántos registros de cada fuente fueron incluidos en el sistema prompt del asistente (pub=826, anid=24, orcid=48, etc.). Útil para auditar el contexto disponible. \\
  \bottomrule
\end{longtable}

\section*{v0.3 --- Modularización y nuevas fuentes de datos}

Las siguientes mejoras fueron implementadas durante la sesión de desarrollo de marzo 2026,
elevando el observatorio de TRL 4 a TRL 5.

\subsection*{1. Modularización del Dashboard (Task 4)}

\textbf{Problema:} \texttt{app.py} tenía 5.275 líneas, haciendo el mantenimiento difícil
y los errores difíciles de aislar.

\textbf{Solución:} Se creó el paquete \texttt{Dashboard/sections/} con 12 archivos:

\begin{itemize}
  \item \texttt{sections/shared.py} (867 líneas) --- constantes, helpers, funciones de UI
  \item \texttt{sections/panel\_indicadores.py} --- KPIs y alertas de gobernanza
  \item \texttt{sections/produccion\_cientifica.py} --- publicaciones, cuartiles, DOI
  \item \texttt{sections/redes\_colaboracion.py} --- red de colaboración y mapa
  \item \texttt{sections/vigilancia\_tecnologica.py} --- noticias, arXiv, INIS, tópicos
  \item \texttt{sections/financiamiento\_id.py} --- ANID, IAEA TC, convenios
  \item \texttt{sections/convocatorias\_matching.py} --- scoring y matching institucional
  \item \texttt{sections/transferencia\_portafolio.py} --- patentes, DataCite, OpenAIRE
  \item \texttt{sections/modelo\_gobernanza.py} --- registro de entidades, ROR
  \item \texttt{sections/formacion\_capacidades.py} --- capital humano
  \item \texttt{sections/asistente\_id.py} --- asistente LLM + exportación PDF
  \item \texttt{sections/\_\_init\_\_.py} --- paquete Python
\end{itemize}

\textbf{Patrón de despacho} en \texttt{app.py} (líneas 1601--1618):
\begin{lstlisting}[style=pythonstyle]
_SECTION_MAP = {
    "Panel de Indicadores":       panel_indicadores.render,
    "Produccion Cientifica":      produccion_cientifica.render,
    # ... 8 secciones mas ...
}
_render_fn = _SECTION_MAP.get(seccion)
if _render_fn is not None:
    _render_fn(_ctx)
\end{lstlisting}

Cada \texttt{render(ctx)} recibe un diccionario con todos los dataframes y contexto.
El código legacy está preservado bajo \texttt{if False:} para rollback fácil.

\subsection*{2. Grafo de Citas OpenAlex (Task 1)}

Nuevo script \texttt{Scripts/fetch\_openalex\_citations.py} que:
\begin{itemize}
  \item Procesa los 877 papers CCHEN en OpenAlex de forma incremental
  \item Obtiene \texttt{cited\_by\_count}, \texttt{referenced\_works} y \texttt{cited\_by\_api\_url}
  \item Descarga hasta 50 papers citantes externos por trabajo CCHEN
  \item Extrae instituciones citantes (top 5 por paper)
  \item Guarda estado en \texttt{citation\_graph\_state.json} para reanudar
  \item Salidas: \texttt{cchen\_citation\_graph.csv} + \texttt{cchen\_citing\_papers.csv}
\end{itemize}

\subsection*{3. Patentes INAPI Chile (Task 2)}

Nuevo script \texttt{Scripts/fetch\_inapi\_patents.py} que:
\begin{itemize}
  \item Busca en \texttt{patentes.inapi.cl/busca} por titular (CCHEN, variantes)
  \item Busca por palabras clave nucleares (reactor, radiofármaco, dosimetría, etc.)
  \item Fallback a CSV abierto de \texttt{datos.inapi.cl} si la API no responde
  \item Filtra por relevancia nuclear con \texttt{\_is\_relevant()}
  \item Salida: \texttt{Data/Patents/cchen\_inapi\_patents.csv}
  \item Genera CSV vacío con headers si no hay resultados (evita fallo en data\_loader)
\end{itemize}

\subsection*{4. Semantic Scholar (Task 3)}

El script existente \texttt{Scripts/fetch\_semantic\_scholar.py} fue activado y ejecutado

\section*{v0.4 --- pgvector, RAG avanzado y marco de evaluación}

Las siguientes mejoras fueron implementadas en la segunda quincena de marzo 2026, consolidando el sistema en TRL 5 con capacidades de IA evaluables.

\subsection*{1. Migración DIAN a Supabase}

La tabla \texttt{dian\_publications} (133 registros internos CCHEN) fue migrada a Supabase via \file{Database/migrate\_dian.py} con:
\begin{itemize}
  \item Schema actualizado en \file{Database/schema.sql} (nueva tabla con RLS público)
  \item Fallback robusto en \file{data\_loader.py}: intenta Supabase, cae a CSV local si falla
  \item Integración verificada en \texttt{sections/produccion\_cientifica.py}
\end{itemize}

\subsection*{2. Embeddings semánticos con columna abstract en Supabase}

\begin{itemize}
  \item Se verificó que \file{cchen\_embeddings\_meta.csv} ya contenía la columna \texttt{abstract} (877 filas)
  \item Se regeneraron los embeddings para asegurar consistencia
  \item Se subieron a Supabase via \file{Database/migrate\_embeddings.py}: 877 vectores en 18 batches de 50, \textbf{0 errores}
  \item La tabla \texttt{paper\_embeddings} ahora incluye: \texttt{openalex\_id, doi, title, year, abstract, embedding vector(384)}
  \item La función RPC \texttt{match\_papers()} fue actualizada para retornar el abstract junto con cada resultado
\end{itemize}

\subsection*{3. Logo institucional CCHEN 360°}

\begin{itemize}
  \item Archivo \file{Dashboard/assets/logo\_cchen360.png} (107 KB) creado
  \item \textbf{Login page:} logo centrado (220px) sobre el título, con fallback a texto si el archivo no existe
  \item \textbf{Sidebar:} logo (160px) reemplaza el texto de encabezado; fallback a texto
  \item Implementado en \file{Dashboard/app.py} con \texttt{\_LOGO\_PATH = Path(\_\_file\_\_).parent / ``assets'' / ``logo\_cchen360.png''}
\end{itemize}

\subsection*{4. Reranking por perfil de medicina nuclear (\texttt{semantic\_search.py})}

\begin{itemize}
  \item Detección automática de consultas de medicina nuclear por keywords
  \item Expansión del pool de candidatos a \texttt{max(top\_k, 5*top\_k, 15)}
  \item Función \texttt{\_profile\_bonus(title, abstract)}: score adicional por términos fuertes (PET, 177Lu, DOTATATE, etc.), secundarios (SPECT, imaging) y penalización por términos negativos
  \item Aplicado tanto en la ruta local (.npy) como en la ruta Supabase pgvector
\end{itemize}

\subsection*{5. Extracción de \texttt{\_build\_assistant\_system\_prompt}}

\begin{itemize}
  \item Función extraída de \texttt{render()} en \file{sections/asistente\_id.py} hacia nivel módulo
  \item Firma: \texttt{\_build\_assistant\_system\_prompt(ctx: dict, patents\_key: str = "") -> tuple}
  \item Reutilizable fuera de Streamlit (sin dependencias de \texttt{st.secrets} ni \texttt{@st.cache\_data})
  \item Retorna \texttt{(system\_prompt\_str, context\_trace\_dict)} con 17 claves de recuentos
  \item Permite evaluación batch desde scripts de línea de comandos
\end{itemize}

\subsection*{6. Marco de evaluación del Asistente I+D}

\textbf{Plantilla de evaluación} (\file{Docs/reports/assistant\_eval\_template.csv}):
\begin{itemize}
  \item 13 consultas Q01--Q13 con campos: \texttt{evaluation\_mode}, \texttt{expected\_data\_sources}, \texttt{must\_include\_keywords}
  \item Q01--Q10: consultas estratégicas (\texttt{structured\_or\_hybrid})
  \item Q04, Q06, Q11--Q13: consultas de recuperación semántica (\texttt{publication\_rag})
\end{itemize}

\textbf{Script de evaluación por lotes} (\file{Scripts/assistant\_eval\_batch.py}):
\begin{itemize}
  \item CLI con \texttt{--evaluation-mode} (\texttt{all | publication\_rag | structured\_or\_hybrid})
  \item Métricas de retrieval: top1\_score, avg\_score, keyword\_hits, n\_with\_abstract
  \item Métricas estructuradas: disponibilidad de fuentes, ratio de cobertura
  \item Genera CSV con columnas vacías para revisión humana
\end{itemize}

\textbf{Script de evaluación con respuestas LLM}\\ (\file{Scripts/assistant\_eval\_structured\_responses.py}):
\begin{itemize}
  \item Llama a Groq API (sin streaming) para generar respuestas reales
  \item Mide latencia (\texttt{response\_ms}): promedio $\sim$35 segundos por consulta (sistema prompt $\sim$22.770 chars)
  \item Columnas heurísticas: \texttt{heuristic\_sources\_mentioned}, \texttt{heuristic\_citation\_tags}
  \item v1 completada (8 consultas, \texttt{llama-3.3-70b-versatile})
  \item v2 en progreso (limita 100K TPD Groq tier gratuito por organización)
\end{itemize}
