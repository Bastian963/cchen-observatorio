# Propuesta de campos — Formulario de ingreso al Observatorio CCHEN

## Objetivo

Definir un formulario claro, explicativo y suficientemente estructurado para que una unidad pueda registrar una necesidad, una oportunidad, una idea o información útil para el Observatorio antes de su evaluación interna.

La lógica del formulario no es solo recibir pedidos. También debe permitir capturar señales tempranas, antecedentes, propuestas y conocimiento que hoy suelen quedar dispersos en correos, reuniones o conversaciones informales.

## Principio de diseño

- El formulario debe ser comprensible para una persona no técnica.
- Cada campo debe explicar para qué sirve.
- Debe permitir tanto pedir algo concreto como compartir una idea, un antecedente o información relevante.
- La evaluación detallada y la priorización final deben quedar del lado del Observatorio.

## Estructura sugerida del formulario

### 1. Identificación de la solicitud

Estos campos permiten saber quién ingresa la solicitud y desde qué unidad.

- Nombre o título de la solicitud
  - Ejemplo: "Indicador de convenios activos por país".
- Unidad solicitante
  - Ejemplo: DGIn, Vigilancia Tecnológica, Transferencia, Dirección.
- Nombre de quien registra la solicitud
- Cargo o rol
  - Opcional, pero útil para entender el contexto de uso.
- Correo de contacto
- Fecha de ingreso

## 2. Tipo de ingreso

Este bloque ayuda a distinguir si se trata de una necesidad operativa, una mejora o una idea que conviene evaluar.

- Tipo de solicitud o aporte
  - Nuevo dato
  - Actualización de dato existente
  - Nuevo indicador
  - Nueva visualización
  - Reporte o exportable
  - Automatización
  - Piloto operativo
  - Idea o propuesta de mejora
  - Información útil para compartir con el Observatorio
  - Otro
- ¿La solicitud corresponde a algo que falta hoy o a una mejora sobre algo ya existente?
  - Falta algo nuevo
  - Hay algo existente que debe corregirse o ampliarse
  - Es una idea para evaluar más adelante

## 3. Descripción de la necesidad o aporte

Este bloque debe ser el más explicativo del formulario. La persona debe poder escribir en lenguaje simple qué necesita y por qué.

- Describa brevemente la necesidad, problema, idea o información que desea compartir
  - Instrucción sugerida: "Explique en pocas líneas qué detectó, qué falta, qué oportunidad ve o qué información cree que conviene incorporar al Observatorio".
- ¿Para qué serviría esto dentro de su unidad?
  - Instrucción sugerida: "Explique el uso esperado. Por ejemplo: seguimiento, reporte a jefatura, priorización, monitoreo, apoyo a decisiones".
- ¿Qué decisión, proceso o tarea ayudaría a mejorar?
  - Instrucción sugerida: "Indique si esto ayuda a decidir, planificar, reportar, hacer seguimiento, activar oportunidades o resolver un problema operativo".
- ¿Qué resultado espera obtener?
  - Ejemplo: un nuevo indicador, una tabla descargable, una alerta, una vista comparativa, una mejora de calidad de datos.

## 4. Campo especial para ideas o información que la unidad quiera aportar

Este campo se agrega explícitamente para no limitar el formulario a requerimientos cerrados.

- Ideas, antecedentes o información que pueda compartir con el Observatorio
  - Instrucción sugerida: "Si no está pidiendo algo concreto, pero tiene una idea, sugerencia, hallazgo, archivo, contacto, contexto o información que podría ser útil para el Observatorio, descríbalo aquí".
  - Ejemplos:
    - "Tenemos una planilla que podría servir como fuente base".
    - "Hay una necesidad recurrente en reuniones de jefatura".
    - "Conviene incorporar seguimiento de una línea temática específica".
    - "Podemos compartir un archivo, base o contacto para profundizar".

## 5. Contexto y evidencia disponible

Este bloque permite saber si ya existe material para evaluar o implementar la solicitud.

- Fuente, archivo, enlace o insumo disponible
  - Instrucción sugerida: "Indique si ya existe una planilla, base de datos, documento, enlace web, presentación, informe o correo relacionado".
- ¿Cuenta con evidencia inicial o antecedentes para respaldar esta solicitud?
  - Sí
  - No
  - Parcialmente
- Describa brevemente la evidencia disponible
  - Ejemplo: nombre de archivo, sistema de origen, enlace, persona contacto, acta previa.
- Área temática asociada
  - Ejemplo: financiamiento, publicaciones, capital humano, colaboración, transferencia, vigilancia, gobernanza.
- Frecuencia esperada de actualización
  - Una vez
  - Semanal
  - Mensual
  - Trimestral
  - Bajo demanda
  - No lo sé todavía

## 6. Prioridad y oportunidad

Este bloque ayuda a ordenar la revisión sin exigir a la unidad una priorización técnica completa.

- Nivel de urgencia percibido
  - Alta
  - Media
  - Baja
- ¿Existe una fecha en que esta información o mejora sería especialmente útil?
  - Campo de fecha o texto libre.
- Impacto esperado si esto se resuelve
  - Alto
  - Medio
  - Bajo
- ¿A quién beneficiaría principalmente?
  - Una persona
  - Un equipo
  - Una unidad
  - Varias unidades
  - Dirección / jefatura

## 7. Evaluación inicial Observatorio

Campos para completar internamente después de la recepción:

- Estado de revisión
  - Recibida
  - En revisión
  - Aceptada
  - En backlog
  - En piloto
  - Implementada
  - Cerrada
  - Fuera de alcance
- Clasificación interna
  - Datos
  - Indicadores
  - Visualización
  - Automatización
  - Gobernanza
  - Piloto operativo
  - Idea para evaluar
  - Información de contexto
- Responsable Observatorio
- Prioridad interna
- Fecha de revisión
- Comentarios de evaluación

## Campos mínimos recomendados para piloto

Si se quiere partir con un formulario liviano, estos serían los mínimos:

- Unidad solicitante
- Nombre de quien registra
- Correo
- Tipo de solicitud o aporte
- Descripción breve de la necesidad, idea o información compartida
- Uso esperado
- Evidencia o insumo disponible
- Urgencia
- Ideas, antecedentes o información que pueda compartir con el Observatorio

## Recomendación de uso

- Mantener lenguaje simple y no técnico.
- Incluir texto de ayuda breve en cada bloque.
- Permitir respuestas abiertas para captar mejor el contexto.
- Dejar la evaluación detallada para el registro interno del Observatorio.
- Revisar las solicitudes en una cadencia fija semanal o quincenal.

## Estado del documento

- Propuesta conceptual.
- No implica implementación inmediata.
- Lista para revisión y validación con jefatura.
